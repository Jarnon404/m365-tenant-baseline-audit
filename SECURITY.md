# Security policy

This repository is public-safe and must not contain customer, tenant or production secrets.

## Reporting issues

Please do not open public issues containing tenant IDs, domains, user principal names, device names, IP addresses, exports or screenshots from real environments.

Use sanitized examples only.

## Data handling

Generated audit reports are local outputs. Review and sanitize them before sharing or committing anywhere.

## Supported versions

The latest release is supported for public examples and repository validation.
